# Range Trading — стратегия и индикатор (TradingView / Pine Script v6)

Торговля в диапазоне (флэте): диапазон определяется на старшем таймфрейме (**1ч**),
входы исполняются на рабочем таймфрейме (**15м**) по касанию границы + RSI +
подтверждающая свеча.

В репозитории два скрипта на **Pine Script v6**:

| Файл | Тип | Для чего |
|------|-----|----------|
| [`pine/range_trading_indicator.pine`](pine/range_trading_indicator.pine) | `indicator` | Визуал: рисует границы диапазона, метки сигналов LONG/SHORT, статус-панель, алерты. Для ручной торговли. |
| [`pine/range_trading_strategy.pine`](pine/range_trading_strategy.pine) | `strategy` | Исполняет входы/стопы/тейки. Даёт метрики бэктеста во встроенном **Strategy Tester** (winrate, profit factor, просадка, число сделок). |

Оба скрипта используют **одинаковые входные параметры и одну и ту же логику
сигналов**, поэтому индикатор и стратегия совпадают 1:1.

## Логика

**Диапазон (1ч):** `upper` = максимум High, `lower` = минимум Low за последние
`N` закрытых часовых свечей (по умолчанию `N = 20`). Мид = центр диапазона.

**Фильтр флэта (опционально):** сделки берутся, только если ширина диапазона в %
от цены попадает в коридор `[min, max]` — это отсекает тренды (слишком широкий
диапазон) и слишком узкие диапазоны, которые съедает комиссия.

**Входы (15м):**

- **LONG** от нижней границы, если:
  - цена в зоне касания нижней границы (`low ≤ lower + touch`),
  - `RSI < 30` (перепроданность),
  - бычья свеча (`close > open`) с телом не меньше заданной доли диапазона свечи.
- **SHORT** от верхней границы, если:
  - цена в зоне касания верхней границы (`high ≥ upper − touch`),
  - `RSI > 70` (перекупленность),
  - медвежья свеча (`close < open`) с достаточным телом.

**Стоп:** чуть за границей диапазона — `lower − buffer` для лонга,
`upper + buffer` для шорта. Буфер = % от высоты диапазона (по умолчанию 20%) или
`ATR × множитель`.

**Тейк-профит:** центр диапазона (консервативно, по умолчанию) или
противоположная граница.

> Пример из ТЗ (BTC/USDT): диапазон 60–70, цена тестит 60 при RSI ~25 и сильной
> бычьей свече → лонг; тейк к центру (65) или к верхней границе (70), стоп ниже 58.

## Установка

1. Открой график **BTC/USDT** на таймфрейме **15 минут**.
2. TradingView → **Pine Editor** → вставь содержимое нужного `.pine` файла →
   **Add to chart**.
3. Проверь, что в настройках **Range timeframe = 60** (1 час).

## Ключевые параметры

| Параметр | По умолч. | Смысл |
|----------|-----------|-------|
| Range timeframe | `60` | Таймфрейм для расчёта диапазона (1ч). |
| Range lookback | `20` | Сколько закрытых часовых свечей берём для max/min. |
| Use only closed HTF candles | `on` | Считать по закрытым часовым свечам — диапазон не перерисовывается внутри бара. |
| RSI length / OS / OB | `14 / 30 / 70` | Период RSI и пороги перепроданности/перекупленности. |
| Boundary touch zone, % | `15` | Насколько близко к границе должна подойти цена, чтобы взвести вход. |
| Min candle body | `0.30` | Требование к «решительной» свече: тело ≥ доли её High-Low. |
| Take-profit target | `Range mid` | Цель тейка: центр диапазона или противоположная граница. |
| Stop buffer | `20% / ATR` | Отступ стопа за границу диапазона. |

## Бэктест и метрики

Добавь **стратегию** на график → вкладка **Strategy Tester**:

- **Overview** — кривая доходности, net profit, max drawdown.
- **Performance Summary** — profit factor, win rate, средний выигрыш/проигрыш.
- **List of Trades** — каждая сделка со стопом и тейком.

Настрой комиссию (`commission_value`, по умолчанию 0.05%) под своего брокера и
проверь на нужном историческом периоде перед реальной торговлей.

## Заметки о перерисовке (repaint)

При включённом `Use only closed HTF candles` диапазон строится по последним `N`
**закрытым** часовым свечам, поэтому уровни не «прыгают» внутри 15-минутного бара.
Сигналы срабатывают на закрытии бара (в стратегии — `process_orders_on_close`),
как и должно быть для честного бэктеста.

## Дисклеймер

Скрипты предоставлены в образовательных целях. Это не финансовая рекомендация.
Любая торговля сопряжена с риском; тестируйте на истории и на демо-счёте.
