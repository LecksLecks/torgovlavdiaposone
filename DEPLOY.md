# Как опубликовать сайт REELCRAFT на своём домене

Сайт — это один файл `index.html`, полностью самодостаточный (весь дизайн внутри,
никаких зависимостей). Его можно залить куда угодно. Ниже — самый простой
бесплатный способ и альтернативы.

---

## Шаг 0. Перед публикацией — подставьте свои данные

Откройте `index.html` и замените:

1. **Ссылки на оплату** — внизу файла, блок `CHECKOUT`:
   ```js
   var CHECKOUT = {
     start: "https://ВАША-ССЫЛКА-ОПЛАТЫ/start",
     pro:   "https://ВАША-ССЫЛКА-ОПЛАТЫ/pro",
     max:   "https://ВАША-ССЫЛКА-ОПЛАТЫ/max"
   };
   ```
   Вставьте ссылки из вашего платёжного сервиса (см. «Как принимать оплату» ниже).
2. **Название бренда** `REELCRAFT` → на своё (поиском по файлу, заменить все).
3. **E-mail в подвале** `hello@reelcraft.ru` → на свой.
4. При желании — цифры в тарифах и текст под себя.

---

## Способ A (рекомендую) — Netlify Drop, без регистрации хостинга и кода

1. Зайдите на **https://app.netlify.com/drop**
2. Перетащите файл `index.html` (или папку с ним) прямо в окно браузера.
3. Через 10 секунд сайт уже онлайн по адресу вида `random-name.netlify.app`.
4. Дальше подключаем свой домен (см. «Шаг с доменом»).

## Способ B — Cloudflare Pages (быстро + бесплатный SSL + свой домен внутри Cloudflare)

1. Заведите аккаунт на **https://dash.cloudflare.com** → раздел **Workers & Pages** → **Create → Pages**.
2. Выберите **Upload assets**, загрузите `index.html`.
3. Нажмите **Deploy**. Сайт готов на `имя.pages.dev`.
4. Вкладка **Custom domains** → добавьте свой домен (если домен куплен в Cloudflare — подключится в один клик).

## Способ C — GitHub Pages (файл уже в этом репозитории)

1. В настройках репозитория на GitHub: **Settings → Pages**.
2. **Source** → выберите ветку (`main` или вашу) и папку `/root`.
3. Сохраните. Через минуту сайт будет на `ваш-логин.github.io/имя-репо`.
4. В том же разделе **Custom domain** введите свой домен и нажмите Save.

## Шаг с доменом (одинаково для всех способов)

### 1. Купите домен
Регистраторы: **Cloudflare Registrar** (по себестоимости), **reg.ru**, **Namecheap**,
**Timeweb**. Хорошее имя для этой ниши: `reelcraft.ru`, `reels-scripts.ru`,
`scenarii.pro` и т.п. Стоимость ~200–1200 ₽/год.

### 2. Подключите домен к хостингу
В панели хостинга (Netlify / Cloudflare / GitHub) найдите раздел
**Custom domain / Add domain**, введите свой домен. Хостинг покажет, какие
DNS-записи прописать. Обычно это одно из двух:

- **CNAME** для `www` → на адрес хостинга (например `random-name.netlify.app`), и
- **A / ALIAS** для корня `@` → на IP, который выдаст хостинг,

либо (для Netlify/Cloudflare проще всего) — сменить **NS-серверы** домена на
серверы хостинга: тогда всё настроится автоматически.

### 3. Пропишите DNS у регистратора
В панели регистратора домена откройте **DNS / Управление зоной** и добавьте
записи, которые показал хостинг. Сохраните.

### 4. Подождите обновления DNS
От 5 минут до нескольких часов (обычно ~15–30 мин). После этого сайт откроется
по вашему домену.

### 5. Включите HTTPS
На Netlify / Cloudflare / GitHub Pages **SSL-сертификат выпускается автоматически
и бесплатно** — просто дождитесь статуса «Active/Secured». Отдельно ничего
покупать не нужно.

## Как принимать оплату за цифровой товар

Выберите сервис, получите ссылку на оплату и вставьте её в блок `CHECKOUT`:

| Сервис | Кому подходит | Плюс |
|--------|---------------|------|
| **Gumroad** | продажи на зарубежную аудиторию | сам отдаёт файл после оплаты |
| **Lava.top / Payhip** | СНГ + цифровые товары | простая выдача, карты РФ |
| **ЮKassa / Продамус** | приём карт в РФ | популярно, чеки самозанятого |
| **Tilda + приём оплат** | если захотите расширить лендинг | конструктор + касса |

Логика простая: в сервисе создаёте товар и получаете **ссылку оплаты** → эту
ссылку вставляете в `CHECKOUT` для нужного тарифа → после оплаты сервис
автоматически выдаёт покупателю файлы/доступ.

> Совет: сами файлы товара (Notion-база, PDF, Google Docs) держите в облаке
> (Google Drive / Notion), а покупателю выдавайте ссылку — это делает большинство
> платёжных сервисов автоматически.
